## ISSUES FOUND

### ❌ FAIL — Check 2: FAQPage JSON-LD missing from static HTML

**What was found:**
The static HTML served to crawlers contains ONE JSON-LD block (in `index.html`, hardcoded) with `@type: ["Organization", "LocalBusiness"]` and `WebSite`. No FAQPage block is present.

**Why it's missing:**
`HomePage.jsx` defines a `faqSchema` with `@type: "FAQPage"` and injects it via React Helmet:
```jsx
<script type="application/ld+json">{JSON.stringify(faqSchema)}</script>
```
The prerender pipeline successfully handles `<meta>` and `<title>` tags from Helmet (confirmed — /trade-advisory and /contact have correct route-specific meta tags). However, `<script>` tags added by Helmet are **not** being injected into the prerendered static HTML. FAQPage only becomes visible after JavaScript hydration in the browser — search engines and AI crawlers doing HTTP-only fetches never see it.

**What was expected:**
At least one JSON-LD block with `@type: "FAQPage"` in the static HTML response.

**Proposed fix (requires your approval before implement):**
Add the FAQPage JSON-LD directly to `index.html` as a second hardcoded `<script type="application/ld+json">` block, just like the LocalBusiness block. This is the most reliable approach — independent of Helmet and prerender.

## OBSERVATION — AI Crawler Blocking in robots.txt (not a check fail, but notable)

The robots.txt contains Cloudflare-managed rules that block several major AI/LLM crawlers:

```
User-agent: ClaudeBot      → Disallow: /
User-agent: GPTBot         → Disallow: /
User-agent: Google-Extended → Disallow: /
User-agent: Amazonbot      → Disallow: /
User-agent: meta-externalagent → Disallow: /
User-agent: Bytespider, CCBot, Applebot-Extended → Disallow: /
```

`Content-Signal: search=yes, ai-train=no` — this allows Google/Bing search indexing but tells AI systems the content may not be used for training.

**Implication:** These blocks prevent AI Overviews (Google SGE), ChatGPT web search, Claude.ai web search, and Perplexity from indexing your content. If you want AI search visibility (AI Overviews, AI-generated answers that cite you), these crawlers need to be allowed. This is a Cloudflare-managed setting — **not automatically adjustable via robots.txt editing** — it requires changing the Cloudflare "AI bots" setting in your Cloudflare dashboard.

This is a business decision, not a mechanical defect. Flagging for your awareness.
