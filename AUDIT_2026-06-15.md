## ISSUES FOUND

### ISSUE 1 — Per-route meta tags: all three routes return identical title/description (FAIL)
**Found:** All three URLs return the same title, description, canonical, and og:url.  
**Expected:** Unique title and description per route.  
**Root cause — two compounding problems:**
1. **Stale deployment:** Production is serving old HTML. Source at HEAD has the correct homepage title `"FCL Freight Forwarding from India | Sattva Global Logistics"` but it hasn't been deployed. Run `.\push.bat`.
2. **React Helmet is client-side only:** Per-route tags set via `<Helmet>` in `TradeAdvisoryPage.jsx` only apply after JS executes. Static crawlers and basic AI crawlers see the index.html values for every route.
**Immediate fix:** Deploy. For a complete fix: evaluate prerendering (vite-plugin-prerender or react-snap) to generate per-route static HTML at build time.

---

### ISSUE 2 — FAQPage JSON-LD missing from static HTML (FAIL → AUTO-FIXED)
**Found:** Static index.html contained only `LocalBusiness` and `WebSite` schema. `FAQPage` was set via Helmet in `HomePage.jsx` — client-side only, invisible to basic crawlers.  
**Expected:** Both `LocalBusiness` and `FAQPage` in static HTML.  
**Fix applied:** Added `FAQPage` JSON-LD block directly to `index.html` with 5 questions. Now visible to all crawlers regardless of JS execution.

---

### ISSUE 3 — theme-color = #2907cc in production (FAIL)
**Found:** Production fetch returns `meta-theme-color: #2907cc`.  
**Expected:** `#050A30`  
**Root cause:** Stale cached HTML in production. Source at HEAD has `content="#050A30"` — confirmed via `git show HEAD:index.html`. origin/main is at the same commit as HEAD, so Vercel or Cloudflare CDN is serving a cached response from a prior deployment.  
**Fix:** Run `.\push.bat`. If it still shows #2907cc after deployment, purge Cloudflare cache: Dashboard → Caching → Configuration → Purge Everything.

---

### ISSUE 4 — /llms.txt served as SPA HTML (FAIL → AUTO-FIXED)
**Found:** `/llms.txt` returned `Content-Type: text/html` — the Vercel catch-all rewrite `/(.*) → /index.html` was intercepting it before the static `public/llms.txt` could be served.  
**Root cause:** `vercel.json` had explicit pass-through rewrites for `/sitemap.xml` and `/robots.txt` but not for `/llms.txt`.  
**Fix applied:** Added `{ "source": "/llms.txt", "destination": "/llms.txt" }` to vercel.json rewrites, before the catch-all. After deployment, `/llms.txt` will correctly serve `text/plain` starting with `# Sattva Global Logistics — AI/LLM Context File`.

---

### ISSUE 5 — /trade-advisory canonical and og:url point to homepage (FAIL)
**Found:** Both canonical and og:url for `/trade-advisory` show `https://www.sattvaglobal.in/` (homepage URL).  
**Expected:** `https://www.sattvaglobal.in/trade-advisory`  
**Root cause:** `TradeAdvisoryPage.jsx` sets the correct canonical via `<Helmet>` (line 172) but this is JS-only. Static HTML always serves index.html's homepage canonical.  
**Note:** Google's crawler executes JS so it WILL see the Helmet-set canonical — this is a concern mainly for AI crawlers that don't run JS. A prerender step would fix this permanently. Not auto-fixed (requires architectural change).

---

### ISSUE 6 — /contact is a permanent 301 redirect to / (WARNING — not auto-fixed)
**Found:** `vercel.json` has `{ "source": "/contact", "destination": "/", "permanent": true }`. There is no `/contact` route in `App.jsx`. Sitemap includes /contact at priority 0.8.  
**Impact:** /contact is not an indexable page — Google sees it as a redirect to /. The sitemap reference is misleading.  
**Options for Sujith to decide:**
- A) Create a `ContactPage` component and `/contact` route in App.jsx (best SEO outcome)
- B) Remove /contact from sitemap.xml and keep the redirect for legacy links
- NOT auto-fixed — business decision required.
